Philo Sign-In Redesign

Improving accessibility and user experience for streaming authentication

Role UX Designer
Timeline 2 months
Tools Figma, User Testing, A/B Testing
Streaming Accessibility Authentication User Testing

The Problem

Philo's existing sign-in flow was causing significant user friction and accessibility barriers. Users were experiencing high drop-off rates during authentication, particularly among users with disabilities and those using assistive technologies.

Key issues identified:

  • Poor contrast ratios that failed WCAG guidelines
  • Complex multi-step authentication process
  • Limited keyboard navigation support
  • Inconsistent error messaging and validation
  • Mobile responsiveness issues

Research & Discovery

I conducted comprehensive user research to understand the pain points in the current sign-in experience and identify opportunities for improvement.

Implementation & Impact

The redesigned sign-in flow was implemented across all Philo platforms, resulting in measurable improvements in user experience and accessibility compliance.

Key implementation highlights:

  • Progressive enhancement approach ensuring compatibility across devices
  • Comprehensive accessibility testing with real users and assistive technologies
  • A/B testing to validate improvements and optimize conversion rates
  • Continuous monitoring and iteration based on user feedback

Reflection

This project reinforced the importance of inclusive design and accessibility-first thinking. By prioritizing users with disabilities, I created a solution that improved the experience for all users.

Key learnings included the value of early accessibility testing, the importance of clear error messaging, and how good accessibility practices often lead to better overall user experience.
